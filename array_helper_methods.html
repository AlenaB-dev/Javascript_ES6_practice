forEach()

The forEach() method is one of the most commonly used array helper methods in JavaScript. It's used to iterate over elements in an array and execute a function for each element.

‚úÖ Use forEach() when:

  • You need to do something with each item (side effects)
  • You don‚Äôt need to return or build a new array

üîπ Syntax

array.forEach((element, index, array) => {
  // your code here
});
  • element: The current item in the array
  • index (optional): The index of the current item
  • array (optional): The array being iterated over

üîπ Example

const fruits = ['apple', 'banana', 'cherry'];

fruits.forEach((fruit, index) => {
  console.log(`${index}: ${fruit}`);
});

Output:
0: apple
1: banana
2: cherry

üîπ Key Features

  • Does not return a new array (unlike map(), filter())
  • Does not stop early (unlike a for loop with break)
  • Mostly used for side effects (e.g., logging, updating the DOM, etc.)

üîπ Real-World Use Case

const users = [
  { name: 'Alice', age: 25 },
  { name: 'Bob', age: 30 },
];

users.forEach(user => {
  console.log(`${user.name} is ${user.age} years old.`);
});

üîπ Common Mistakes

‚ùå Trying to return values: forEach() doesn‚Äôt return anything useful ‚Äî use map() if you want to create a new array. 

const result = fruits.forEach(fruit => {
  return fruit.toUpperCase();
});

console.log(result); // undefined
map()

The map() method is used to create a new array by applying a function to each element of an existing array. It does not modify the original array.

üîπ Syntax

const newArray = array.map((element, index, array) => {
  // return transformed element
});
  • element ‚Äî the current item in the array
  • index (optional) ‚Äî the index of the current item
  • array(optional) ‚Äî the array map() was called on

üîπ Example

const numbers = [1, 2, 3];
const doubled = numbers.map(num => num * 2);

console.log(doubled); // [2, 4, 6]

üîπ Key Features

  • Returns a new array with the same number of elements.
  • Does not change the original array.
  • Commonly used to transform or extract data.

üîπ Real-World Use Case

const users = [
  { name: "Alice", age: 25 },
  { name: "Bob", age: 30 }
];

const names = users.map(user => user.name);
console.log(names); // ["Alice", "Bob"]

üîπ Common Mistakes

Forgetting to use return inside the callback when using a function block ({}). 

// Incorrect:
numbers.map(function(num) {
  num * 2; // No return, so it returns undefined
});

// Correct:
numbers.map(function(num) {
  return num * 2;
});
filter()

The filter() method creates a new array containing only the elements that pass a test (a function you provide). It does not modify the original array.

üîπ Syntax

array.filter(callback(element, index, array), thisArg)
  • callback: A function that returns true to keep the element, or false to skip it.
  • element: The current element being processed.
  • index (optional): Index of the current element.
  • array (optional): The array being processed.
  • thisArg (optional): Value to use as this inside the callback.

üîπ Example 1: Filter Numbers Greater Than 10

const numbers = [5, 12, 8, 20, 3];

const filtered = numbers.filter(num => num > 10);

console.log(filtered); // [12, 20]

üîπ Example 2: Filter Even Numbers

const numbers = [1, 2, 3, 4, 5, 6];

const evens = numbers.filter(num => num % 2 === 0);

console.log(evens); // [2, 4, 6]

üîπ Example 3: Filter Objects by Property

const users = [
  { name: "Alice", isActive: true },
  { name: "Bob", isActive: false },
  { name: "Charlie", isActive: true }
];

const activeUsers = users.filter(user => user.isActive);

console.log(activeUsers);
/*
[
  { name: "Alice", isActive: true },
  { name: "Charlie", isActive: true }
]
*/
