üî∏ Template literals

Template literals are a feature in JavaScript used for working with strings. They make it easier to:

  1. Insert variables or expressions into strings.
  2. Write multi-line strings.
  3. Improve the readability and flexibility of your code.

üîπ Syntax

Template literals use backticks ( ` ) instead of regular quotes (' or "). 

`This is a template literal`

üîπ 1. Embedding Variables with ${}

You can insert variables or expressions using ${} inside the template literal: 

const name = "Alena";
const age = 30;
const message = `My name is ${name} and I am ${age} years old.`;
console.log(message);
// Output: My name is Alena and I am 30 years old.

üîπ 2. Multi-line Strings

No need to use \n or string concatenation ‚Äî just press Enter:

const poem = `Roses are red,
Violets are blue,
JavaScript is awesome,
And so are you.`;

console.log(poem);

üîπ 3. Expressions Inside Template Literals

You can insert any JavaScript expression inside ${}:

const a = 5;
const b = 10;
const result = `The sum of a and b is ${a + b}`;
console.log(result);
// Output: The sum of a and b is 15

üîπ 4. Function Calls

function greet(name) {
  return `Hello, ${name}!`;
}

console.log(`${greet("Alena")}`); // Hello, Alena!

üî∏ What is a Tagged Template?

A tagged template is a way to customize how a template literal is processed by using a function (called a ‚Äútag‚Äù).
Instead of just creating a string, you pass the template literal to a tag function, which can then manipulate the strings and values however it wants.

üîπ Syntax

tagFunction`String text ${value1} more text ${value2}`;

The tag function receives:

  • an array of string parts (split at ${}),
  • and the interpolated values as arguments.

üîπ Example

function tag(strings, ...values) {
  console.log('Strings:', strings);
  console.log('Values:', values);
}

const name = "Alena";
const age = 30;

tag`My name is ${name} and I am ${age} years old.`;

Output (sql):

Strings: [ 'My name is ', ' and I am ', ' years old.' ]
Values: [ 'Alena', 30 ]

It can be then use ‚Äî for formatting, escaping HTML, translation, logging, etc.

üîπ Example: Custom Formatter

function highlight(strings, ...values) {
  return strings.map((str, i) => {
    return str + (values[i] ? `${values[i]}` : "");
  }).join('');
}

const name = "Alena";
const skill = "JavaScript";

const sentence = highlight`Hi, I'm ${name} and I love ${skill}!`;

console.log(sentence);
// Output: Hi, I'm Alena and I love JavaScript!

üîπ Use Cases

  1. Custom string formatting
  2. Escaping unsafe HTML (e.g., in frameworks)
  3. Localization and translation
  4. Logging/debugging
  5. Styled components in libraries like styled-components (React)
