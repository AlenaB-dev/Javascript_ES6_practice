In JavaScript, object literals are a concise way to create objects. Enhanced object literals, introduced in ECMAScript 2015 (ES6), provide additional syntax features that make object creation even more expressive and readable.

Before ES6, creating objects with object literals involved a more verbose syntax: 

var person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city: 'Anytown',
    state: 'CA'
  }
};

With ES6, you can use the following syntax:

üîπ Syntax

const person = {
  name: 'John',
  age: 30,
  address: {
    street: '123 Main St',
    city,
    state
  }
};

üì¶ Here are some key features of enhanced object literals:

  1. Property Shorthand: When the property name matches the variable name, you can omit the property name and simply use the variable name.
  2. Computed Property Names: You can use expressions to compute property names.
  3. Method Definitions: You can define methods directly in the object literal.
  4. Object Property Value Shorthand: When the property value is an expression, you can omit the assignment operator (=).
  5. Spread Operator: You can use the spread operator (...) to merge objects. 
// Property shorthand
const person = {
  name,
  age,
  address: {
    street,
    city,
    state
  }
};

// Computed property names
const person = {
  [`name${i}`]: `John${i}`,
  age: 30
};

// Method definitions
const person = {
  sayHello() {
    console.log('Hello!');
  }
};

// Object property value shorthand
const person = {
  name: 'John',
  sayHello() {
    console.log(`Hello, ${this.name}!`);
  }
};

// Spread operator
const person = {
  ...defaultPerson,
  name: 'John'
};

// Object destructuring
const { name, age } = person;
