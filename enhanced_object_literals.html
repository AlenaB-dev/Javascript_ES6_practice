Enhanced Object Literals are a set of new, cleaner, and more concise ways to write object properties and methods introduced in ES6 (ES2015). They help make your code easier to read and write.

üîπ Key Features of Enhanced Object Literals:

1. Property Shorthand.
If the property name is the same as the variable name, you can omit the value: 

const name = "Alice";
const age = 25;

// Old way:
const person = { name: name, age: age };

// New shorthand:
const person = { name, age };

2. Method Shorthand.
You can define methods without the function keyword: 

const obj = {
  greet() {
    console.log("Hello!");
  }
};

Instead of:


  const obj = {
  greet: function() {
    console.log("Hello!");
  }
};

3. Computed Property Names.
You can use expressions inside square brackets [] to define property names dynamically: 


  const propName = "score";
const obj = {
  [propName]: 100
};

console.log(obj.score); // 100

4. Defining properties and methods using variables or functions already in scope.

üîπ Example using all features together:


  const prefix = "user";
const id = 123;

const user = {
  id,
  name: "Bob",
  [`${prefix}_id`]: id,
  greet() {
    console.log(`Hi, I'm ${this.name}`);
  }
};

user.greet();        // Hi, I'm Bob
console.log(user.user_id); // 123

üî∏ Why you might get errors when working with object literals:

  • Missing commas between properties.
  • Forgetting to use square brackets for computed property names.
  • Using ES6 features in an environment or context that doesn‚Äôt support them.

üì¶ Summary of key features of enhanced object literals:

  1. Property Shorthand: When the property name matches the variable name, you can omit the property name and simply use the variable name.
  2. Computed Property Names: You can use expressions to compute property names.
  3. Method Definitions: You can define methods directly in the object literal.
  4. Object Property Value Shorthand: When the property value is an expression, you can omit the assignment operator (=).
  5. Spread Operator: You can use the spread operator (...) to merge objects.
