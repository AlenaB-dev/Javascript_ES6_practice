JS engine phases

  • Compilation. Create activation Object with all variables
  • Execution. Execute code line by line

Variable lifecycle phases

  • Declaration. Register in the corresponding scope.
  • Initialization. Allocate memory for the variable. Variable declared with "var" will get automatic value "undefined".
  • Assigment. Assign value to all initialized variable.

Var lifecycle

  • Compilation phase include declaratiob and initialization. Variable is "initialized" and value is "undefined"
  • Execution phase incule assigment. Variable is "assigned" and has "value"

Undeclared variable lifesysle

  • Compilation phase - nothing will happened.
  • Execution phase will include all three phases, 1. Declaration 2> Initialization 3. Assigment, together. Variable is "declared" in the global scope, "initialized", "assigned" and value is "value".

    • *this can happen only without using "use strict" mode.

Let lifecycle

  • Compilation phase. Declaration - temporal dead zone (TDZ) - variable is "declared" but "uninitialized".
  • Execution phase. Initialization - "initialized" with "indefined". Assigment - variable is "assigned" and has "value".

Const lifecycle

  • Declaration phase. Temporal dead zone (TDZ) - variable is "declared" but "uninitialized".
  • Execution phase. Initialization and Assigment - variable is "initialized", "assigned" and has "value".

*TDZ - special interval between start of the code execution until declaration using variables let or const. Accessing variable in the Temporal dead zone will lead to the following error - RefernceError: Variable is not defined.

Function lifecycle

  • Compilation phase. Declaration, Initialization and Assigment all in one stage. Variable is "declared", "initialized", "assigned" and value is "value"
  • Execution phase - variable is ready to use.

What is hoisting?

Hoisting is a term that describes proces of compilation and execution phases.
