Variables scope

Scope refers to the current context of execution in which variables are accessible. In JavaScript, scope determines where variables can be used.

    üî∏ Types of Scope:
  • 1. Global Scope A variable declared outside any function or block is in the global scope and can be accessed anywhere. 
    • 
          let globalVar = 'I am global'; function showVar() {
          console.log(globalVar); // ‚úÖ accessible }
  • 2. Function Scope Variables declared with var inside a function are accessible only inside that function. 
    • function testScope() { var localVar = 'I am local';
          console.log(localVar); // ‚úÖ accessible } console.log(localVar); // ‚ùå
          Error: localVar is not defined
  • 3. Block Scope Variables declared with let or const are only accessible within the block { ... } they are defined in. 
    • 
          if (true) { let blockVar = 'Inside block'; console.log(blockVar); // ‚úÖ
        accessible } console.log(blockVar); // ‚ùå Error

Good to Know: Always prefer let or const over var. Use const if you don‚Äôt plan to reassign the variable. Keeping variables limited to a small scope helps avoid bugs and confusion.

Variable reassigment

    Reassignment means changing the value of a variable after it has been declared.
  • üî∏ Using let Variables declared with let can be reassigned.
    • 
let name = "Alice";
name = "Bob"; // ‚úÖ This is allowed

console.log(name); // "Bob"
  • üî∏ Using const Variables declared with const cannot be reassigned. (Const is variable that can't be reassigned) 
    • 
  const city = "Paris";
city = "London"; // ‚ùå Error: Assignment to constant variable

    üîπ But if a const holds an object or array, the contents of the object or array can be changed:

person.name = "Bob"; // ‚úÖ This is allowed

console.log(person.name); // "Bob"

Variable redeclaration

Variable redeclaration means declaring a variable again using the same name in the same scope. In JavaScript, the behavior depends on whether you're using var, let, or const.

  • ‚úÖ var allows redeclaration
    • 
        var x = 10;
var x = 20; // No error
console.log(x); // 20

    var is function-scoped Can be redeclared and reassigned Not recommended due to possible confusion and bugs

  • ‚ö†Ô∏è let and const do not allow redeclaration in the same scope
    • 
        let y = 5;
// let y = 10; // ‚ùå SyntaxError: Identifier 'y' has already been declared

const z = 3;
// const z = 3; // ‚ùå SyntaxError: Identifier 'z' has already been declared

    let and const are block-scoped Cannot be redeclared in the same block let can be reassigned, but const cannot

Hoisting

Hoisting is JavaScript's default behavior of moving declarations to the top of the current scope (function or global). üí° Only declarations are hoisted, not initializations.

  • ‚úÖ Example with var:
    • 
    console.log(a); // undefined ‚Äî declaration is hoisted, but not the value
var a = 5;

    JavaScript internally treats it like:

    
    var a;       // declaration hoisted
console.log(a); // undefined
a = 5;       // initialization happens later
  • ‚ö†Ô∏è Example with let and const:
    • console.log(b); // ‚ùå ReferenceError: Cannot access 'b' before initialization
let b = 10;

let and const are hoisted, but not initialized. They're in a "temporal dead zone" from the start of the block until the line where they're declared.

Variables usage guadelines

  • Make code readable for other developers
  • Always declare variables before first usage
  • Always add "use strict" at the beggining of each JS file, it makes code more secure
  • Don't expose to the outer scopes locally used variables

Opinion 1. Let is new Var

Use const whenever is possible. In other cases use let.

Opinion2. Let is Var extension

Var - for variables that are used across entire function. Declared usually at the beggining of the scope

Let - for temporal variables that are limited to the block scopes. Declared directly in the block or just before it

Const - for variables that shouldn't be reassigned. Declaration place depends on the const usage.
